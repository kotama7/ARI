# ari-skill-paper-re

MCP skill for **PaperBench-format reproducibility evaluation**
(v0.7.0).  Two phases:

- **Phase 1 — sandbox execution**: fetch the paper's code bundle,
  generate a `reproduce.sh`, and run it inside a sandbox under a
  hard wall-time cap.
- **Phase 2 — grading**: feed the run's outputs through PaperBench
  `SimpleJudge`, scored against a rubric (typically generated by
  `ari-skill-replicate`).

## MCP tools

| Tool | Purpose |
|---|---|
| `fetch_code_bundle` | Fetch + verify a code bundle by `ref` + `sha256` |
| `build_reproduce_sh` | LLM writes `reproduce.sh` from paper text + rubric |
| `run_reproduce` | Run the script in a sandbox (SLURM / Apptainer / Docker / local), bounded by `ARI_REPLICATOR_TIME_LIMIT_SEC` |
| `grade_with_simplejudge` | LLM grades the outputs against the rubric leaves |

`fetch_code_bundle` is purely deterministic (digest-verified
download).  `run_reproduce` is wall-time-non-deterministic but
process-bounded.  `build_reproduce_sh` and `grade_with_simplejudge`
are LLM-driven, hence P2 exceptions.

## End-to-end flow

```
ors_generate_rubric  (replicate-skill)    → ors_rubric.json + ors_rubric.meta.json
ors_seed_sandbox     (this skill)          → repro_sandbox/{reproduce.sh, code/...}
ors_build_reproduce  (this skill)          → repro_sandbox/{reproduce.sh, source files}
ors_run_reproduce    (this skill)          → ors_phase1.json   (Phase 1: sandbox-execute reproduce.sh)
ors_grade            (this skill)          → ors_grade.json    (Phase 2: SimpleJudge over the rubric leaves)
```

The flow is what replaced the v0.6.0 `react_driver`-based
reproducibility check with a PaperBench-aligned pipeline.

## Environment variables

| Variable | Purpose | Default |
|---|---|---|
| `ARI_PAPERBENCH_PATH` | Override the bundled `vendor/paperbench/` path | `vendor/paperbench/` |
| `ARI_MODEL_REPLICATOR` | LLM that drives `build_reproduce_sh` | falls back to `ARI_LLM_MODEL` |
| `ARI_LLM_MODEL` | Grader LLM | (none) |
| `ARI_REPLICATOR_TIME_LIMIT_SEC` | Wall-time cap for `run_reproduce` | `43200` (12 h) |

Vendored bridge modules (`nanoeval`, `preparedness_turn_completer`)
are exposed via `src/_vendor_path.py`; do not import from them
directly outside this skill.

## ORS / EAR relationship (v0.7)

PaperBench grading is the heart of the **ORS (Object Repository
Spec)** reproducibility chain.  Together with the EAR bundle
produced by `ari-skill-transform`, it turns "does the paper
reproduce?" into a tractable pipeline that can be re-run on any
host that can clone the bundle.

See `docs/concepts/publication-lifecycle.md#publication-lifecycle-v070` for the
publication / verification lifecycle and
`ari-skill-replicate/README.md` for the upstream rubric generator.

## Vendored dependencies

The skill bundles PaperBench under `vendor/paperbench/` (git
submodule).  Licensing follows the upstream PaperBench repository.
Update the submodule independently of ARI core releases — only the
skill's compatibility shim guarantees go through `ari.public.*`.
