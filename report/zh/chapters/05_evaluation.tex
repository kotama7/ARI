\section{评估(暂缓)}\label{chapter:eval}

\textbf{本章有意保持简短。本快照下未进行实际评估。}本报告的早
期草稿曾给出具体的中位数与按叶得分;复审后我们发现那些数字是
说明性的占位数,并非源自冻结的 ARI 检查点的实测,故已
删除。仅保留真实存在的产物。

\emph{已实现}的部分:

\begin{itemize}
  \item 成本核算机制:每次模型调用都被附加美元成本,并按运行
        汇总进检查点的结果记录。
  \item 探索循环经停滞检测器
        (\cref{sec:exploration:termination})终止;停滞而非
        步数 / 成本预算($\Tmax$ / $\Cmax$)触发的经验频率,正是
        本章拟在快照确定后报告的内容。
  \item 本报告中的每幅数据图都是由固定种子生成的确定性合成占位图
        (以保证文档构建可复现),无一作为证据被引用。
\end{itemize}

\emph{未实现}的部分:

\begin{itemize}
  \item 确定 ARI v0.7.2 冻结检查点,并由其真实的搜索树与运行聚合
        (而非当前的合成占位图)导出图表。
  \item 报告中位运行成本、按种子的探索曲线、基于真实评审的按类别
        评分细则分布。
  \item 报告对真实目标论文(两篇对照性的 Supercomputing~2024 投稿
        ——一篇稀疏线性代数、一篇有损压缩内核)的完整 PaperBench
        风格再现得分。 每篇需对真实模型进行数小时的复现 rollout,
        推迟到操作性 v0.8.0 阶段。
\end{itemize}

其他章节中的交叉引用(例如 \cref{sec:exploration:termination}
内的)仅指示经验证据将来安置之处,而非当前所在之处。

v0.7.2 记录了对执行配置路径
(\cref{sec:paper:paperbench:v072})的两项小型 sanity check:
评分单生成对 HPC 论文正确发出携带论文报告规模包络的多 rank
(MPI) 执行配置,并对单机论文省略;且一次多标志集群调度在运行时
probe 剥离该调度器不支持的标志后被本地调度器接受。这些只是验证
接线是否打通,而非度量方法论,未作为证据引用。

操作性 v0.8.0 阶段本身随后在 lossy-compression 对象上得到执行。
它首先暴露了两处一直在悄然剥夺智能体执行得分的框架缺陷:再现阶段
在智能体组装其自包含仓库的目录之外调用了智能体的入口脚本;以及
宿主驻留沙箱缺少一项模型所期望、而容器镜像已具备的文件编辑设施。
在两者均修正后,贯穿全部三个阶段的完整 rollout 生成了可编译的
GPU 原生再现,它获取了真实数据切片并扫掠了一段误差界范围。其余
两项不足属于智能体自身的行为而非框架:智能体在时间预算之内远未
用尽便自行终止,且它计算了一项 GPU 预测的质量指标却将其从报告
结果中省略。与 v0.7.2 的检查相同,这只是端到端地验证流水线而非
度量方法论,该单一对象不作为定量证据引用。
