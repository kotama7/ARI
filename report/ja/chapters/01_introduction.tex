\section{はじめに}\label{sec:intro}

\subsection{動機}\label{sec:intro:motivation}

機械学習および周辺の計算分野における研究ワークフローは、共通の
骨格を共有している:研究アイディアをブレインストームし、実験を
設計・実行し、結果を分析し、論文を執筆し、そして反復する。各
ステップは大規模言語モデル(LLM)にとってますます扱いやすく
なっており、発表済みのシステムは今やこのループの相当な区間を
自動化している。AI~Scientist \cite{lu2024ai,lu2026towards} は
着想から論文までの全サイクルを閉じ、その結果を自動レビューアで
採点する;Agent~Laboratory \cite{schmidgall2025agent} は人手で
与えられた着想を LLM 判定のゲートのもとで実行する;AIDE
\cite{jiang2025aide} はエンジニアリングの内部ループを候補解上の
木探索として定式化し、同一モデルを与えた条件下でこの定式化が
汎用スキャフォルドを凌駕することを MLE-bench \cite{chan2024mle}
が見出した;VirSci \cite{su2024many} は、マルチエージェントの
熟議が単一エージェントよりも新規性の高い研究アイディアを生むこ
とを示すが、アブストラクトで止まる;Story2Proposal
\cite{qian2026story2proposal} は、永続的な構造契約のもとで完成
した実験的証拠を原稿へ変換する;そして PaperBench
\cite{starace2025paperbench} は、エージェントが発表済み論文を
どれだけ忠実に複製するかを事後的に計測する。

これらのシステムのいずれも拘束力をもって扱わないのは、生成され
た論文が\emph{述べる}ことと、その実験が\emph{生み出した}ことと
の関係である。存在する場合でも、検証は助言的にとどまる:
AI~Scientist のレビューアは完成稿を採点する一方、ハルシネー
ション制御はプロンプト水準にとどまる——その著者らは、誤って
報告されたハードウェアと、改善として提示された悪化した指標を
記録し、「報告された結果のより踏み込んだ自動検証」を重要な今後
の課題として挙げている \cite{lu2024ai};Agent~Laboratory の
模擬レビューアは、10 点尺度で人間レビューアのスコアを 2.3 点
過大評価する \cite{schmidgall2025agent};そして PaperBench の
中心的な発見は実行ギャップである——エージェントは書かれた
コードに対しては相当な得点を獲得するが、それを実行して論文の
結果に一致させることに対してはほとんど得点しない
\cite{starace2025paperbench}。

ARI はこのギャップを埋めるために存在する。ユーザは研究問題と
計算予算を与える;システムは、発表可能な論文、各主張を裏付ける
実験コード、そして到達経路の完全な監査記録を返す:探索木の各
ノード、各ツール呼び出し、各モデル呼び出し、そして API 利用の
各ドルが、再現性の単位である単一の\term{チェックポイントディレクトリ}内に記録される。

\subsection{リリース条件としての検証済み主張}\label{sec:intro:verified}

ARI を際立たせる筋は、パイプライン全体を貫いて走る。着想の時点
で、実行は\term{メトリック契約}を鋳造する:最終的な論文が支え
ねばならない反証可能な主張であり、その各々が証拠とみなされる
正確な測定名を明示する。LLM の命名は再生成を生き延びないため、
契約は一度だけ鋳造され凍結される(\cref{sec:paper:contract})。
探索の間、ノードはそれらの名前のもとで測定を発し、その名前が
未充足の主張へ向けて拡張を誘導する。そして\term{系譜連鎖}は、
先行する測定から\emph{計算された}証拠——あるフィット、その
保留検証、モデルに基づく選択——を、繰り返しの probe へ退行する
代わりに単一の系譜に沿って実行させる
(\cref{sec:exploration:lineage-chaining})。執筆の時点では、
あらゆる数値への言及は、それが依拠するノードと測定に結び
つけられ、決定論的な claim--evidence ゲート——ループ内に言語
モデルは存在しない——が、報告された各数値を記録された結果から
再導出する。客観的な虚偽は最終ゲートでリリースを阻止する;
補完的な意味論的レビューの主観的な所見は、改稿ループに助言する
にとどまる。

\subsection{貢献}\label{sec:intro:contrib}

本レポートは四つの貢献を形式化する:

\begin{enumerate}
  \item \textbf{決定論的な claim--evidence ゲート}によって強制
        される\textbf{idea 所有のメトリック契約}
        (\cref{sec:paper:contract}):レビューアによる出力への
        意見ではなく、パイプラインの構造的性質としての検証。
  \item 研究ステップノード上の\textbf{最良優先木探索}(BFTS)
        (\cref{sec:exploration:formalization})であり、その
        選択と拡張は、構造化された候補記述上で LLM 主導であり、
        決定論的なフォールバックランキングを伴う
        (\cref{sec:exploration:node-selection})。
  \item 階層型ルブリック $\Rubric$ をキーに据える\textbf{論文
        生成パイプライン};その葉判官が個々の主張を採点し、
        システムスコアは
        $\paperscore{\paper}=\sum_i \rubricweight{i}\,\judgefn{i}(\paper)$
        となる(\cref{sec:paper:rubric})。PaperBench 複製器は
        ツール呼び出しとして統合される
        (\cref{sec:paper:paperbench})。
  \item \textbf{チェックポイントスコープ}の規律:すべての外部
        状態(メモリ、コスト台帳、系譜ポインタ)は
        \texttt{\$ARI\_CHECKPOINT\_DIR} 配下に書かれ、ユーザの
        ホームディレクトリには決して書き出されない。これは
        \emph{決定性}という設計原則の運用上の帰結であり、
        \cref{sec:overview:principles} が完全に提示する 5 つの
        うちの一つである(本報告では、そこおよび以降を通じて
        この原則を P2 と呼ぶ)。
\end{enumerate}

\subsection{本レポートの適用範囲}\label{sec:intro:scope}

本レポートは \texttt{ari-core} および 14 個の
\texttt{ari-skill-*} パッケージの v0.9.0 を記述する。そのテーマ
は end-to-end の主張検証である:\cref{sec:paper:contract} の
mint-once 契約とゲートの強化;系譜連鎖
(\cref{sec:exploration:lineage-chaining});発表済みの VirSci
熟議エンジンをライブの文献スナップショット上で無改変に駆動する
オプトインの着想生成経路(\cref{sec:related:autonomous});
プラットフォームを認識した着想生成——探られたプラットフォーム
能力の事実を着想生成に折り込み、計画が実行プラットフォームの
提供しない測定を約束できないようにする;そして、それ自体が
ゲート検証済みであり、宣言された反証可能な主張 12 件のすべてが
機械検証済みの証拠を伴うリリース済みのサンプル論文
(\cref{chapter:eval})。これらの追加は v0.8.x の基盤の上に
築かれる——プロトコルに忠実な三段の PaperBench 再現契約
(エージェントロールアウト、再現、採点)、ホスト真実の環境
ガードレール、設定可能な探索評価層;\cref{chapter:eval} で
報告される非可逆圧縮対象に対するオペレーショナル再現パスは、
当時最新の v0.8.0 パイプライン上で実施された。本文はアルゴリ
ズムとデータフローをモジュール水準で記述し;システムが用いる
逐語の LLM プロンプトは \cref{sec:prompts} に列挙する。本版に
おける経験的研究は予備的であり、制御されたベンチマークは今後の
課題として残す。
