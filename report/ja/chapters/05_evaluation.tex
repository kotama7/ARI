\section{評価(保留)}\label{chapter:eval}

\textbf{本章は意図的に短い。本スナップショットでは評価を実施し
ていない。}本レポートの旧草稿は具体的な中央値や葉ごとのスコアを
記載していたが、見直したところ、それらの数値は説明用のプレース
ホルダであり、凍結された ARI チェックポイントからの実測ではない
ことが判明した。そのため、それらは削除した。コードまたは稼働中
インフラに裏付けされた成果物のみを残す。

\emph{実装済み}のもの:

\begin{itemize}
  \item コスト計上機構:各モデル呼び出しにドルコストが付与され、
        実行ごとにチェックポイントの結果レコードへ集計される。
  \item 探索ループは停滞検出器
        (\cref{sec:exploration:termination})により終了する;
        ステップ・コスト予算($\Tmax$ / $\Cmax$)ではなく停滞が
        支配する経験的頻度を本章で報告する予定 — スナップショット
        が揃った時点で。
  \item 本レポートのデータ図はすべて固定シードから生成した決定的
        な合成プレースホルダであり(文書ビルドの再現性のため)、
        いずれも証拠として\emph{引用していない}。
\end{itemize}

\emph{未実装}のもの:

\begin{itemize}
  \item ARI v0.7.2 の凍結チェックポイントを確定し、図を現在の合成
        プレースホルダではなく実際の探索木と実行集約から導出する
        こと。
  \item 中央値の実行コスト、シード別の探索曲線、実レビューに基づく
        カテゴリ別ルブリック分布の報告。
  \item 実対象論文(対照的な 2 篇の Supercomputing~2024 投稿——
        一方は疎線形代数、他方は非可逆圧縮カーネル)に対するフル
        ロールアウトの PaperBench 風再現スコア報告。 各々実モデル
        に対する複数時間の複製ロールアウトを要し、 オペレーショナル
        v0.8.0 パスに持ち越す。
\end{itemize}

他章からの相互参照(例えば \cref{sec:exploration:termination} 内)
は、経験的証拠が将来配置される場所を指し示すのみであり、現在置
かれている場所を指してはいない。

v0.7.2 は実行プロファイル経路
(\cref{sec:paper:paperbench:v072})に対する 2 件の小規模な
サニティチェックを記録している:ルブリック生成が、HPC 論文に
対しては論文の報告スケール包絡を伴う多ランク (MPI) 実行プロファ
イルを正しく発行し、単一マシン論文に対しては省略した;また
マルチフラグのクラスタディスパッチが、ランタイム probe が当該
スケジューラの非対応フラグを落とした後、ローカルスケジューラに
受理された。 これらは配管を試すものであって方法論の測定ではなく、
証拠としては引用していない。

操作性 v0.8.0 パス自体は、その後、lossy-compression 対象に対して
実施された。これはまず、エージェントに実行クレジットを暗黙のうちに
与えそこねていた 2 件のハーネス欠陥を露呈させた:再現ステージが、
エージェントが自己完結リポジトリを組み立てたディレクトリの外で
エージェントのエントリスクリプトを呼び出していたこと、そして
ホスト常駐のサンドボックスが、モデルが期待しコンテナイメージは
既に備えるファイル編集機能を欠いていたことである。両者を是正すると、
3 ステージすべてを通すフルロールアウトが、実データスライスを取得し
誤差限界の範囲を掃引する、コンパイル可能な GPU ネイティブの再現を
生成した。残る 2 つの不足はハーネスではなくエージェント自身の挙動に
属する:エージェントは時間予算の十分内側で自発的に終了し、また
GPU 予測による品質指標を算出しながら、それを報告結果から省いた。
v0.7.2 のチェックと同様、これは方法論ではなくパイプラインを端から
端まで試すものであり、単一対象は定量的証拠として引用しない。
