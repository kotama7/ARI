## 2. 削除要件(中核)

本計画における「削除」を6種に分類し、各々の**前提条件**を定める。
**原則: 内容を持つファイルは「削除」ではなく「移動(`git mv`)」する。** 真の削除は下表に限る。

| ID | 削除対象 | 種別 | いつ | 前提条件(これを満たさなければ削除しない) |
|---|---|---|---|---|
| **D-1** | 移動元の旧パス | 移動(git mv) | U2 / U4 | `git mv` で移動済=内容は移動先に保全。単独 `rm` は禁止 |
| **D-2** | 空になった `howto/` 等のディレクトリ | 空ディレクトリ削除 | U2 / U4 | 配下が**完全に空**(`rmdir` が成功する=空、で安全側に倒す) |
| **D-3** | 一時リダイレクト・スタブ | 一時ファイル削除 | U6(次マイナー) | スタブを設置した場合のみ。当該旧パスへの**被参照ゼロ**を確認後 |
| **D-4** | `assets/` へ集約後の重複メディア | 重複削除 | U5(任意) | 新パスへ**sha256 一致でコピー済**かつ全参照が新パスに追従済 |
| **D-5** | 計画書3点(本書/リファクタリング計画書/ソース対応機構設計書) | **削除** | U6 | scope 完了後に `[plan-deletion]` コミットで**削除**(`refactor_audit.md` §10 の前例)。履歴は `git log --diff-filter=D --follow` で復元可能。削除前に §3 D-5 の double-check を必須 |
| **D-6** | front-matter/本文中の stale ソース参照 | 参照修正 | 各フェーズ随時 | 「ソースが消えたのか移動したのか」を判定。**分割・改名なら新パスへ付け替え(削除ではない)** |

### 削除してはいけないもの(do-not-delete)

- 内容を持つライブ doc(常に移動。削除しない)。
- `_archive/refactor_audit.md` — 歴史的記録として**残す**(冒頭に「historical snapshot」明記済)。
- メディア実体(`logo.png` / `sample_paper.pdf` / `movie/` / `images/`)— サイト/doc の参照を
  追従し終えるまで削除禁止。これらは**公開資産**であり外部リンクの可能性がある。
- **自分(エージェント)が作成していないファイル**、および**説明と中身が矛盾するファイル** —
  削除・上書きせず、まずユーザに報告する(§5 の基本原則)。

## 3. 削除前 double-check 手順(中核)

### 基本原則(全削除に共通・無条件)

> **削除・上書きの直前に、対象を必ず開いて中身を確認する。**
> 期待した内容と違う、説明と矛盾する、または自分が作成したものでない場合は、
> **削除を中止してユーザに報告する**。「計画にあったから」は実行の十分条件ではない。

### 共通チェックリスト(D-1〜D-6 すべてで全項目 ✓ が必須)

```
[ ] 1. git status がクリーン(意図しない変更が混在していない)
[ ] 2. 削除対象を実際に開いて中身を確認した(期待どおりか)
[ ] 3. 被参照ゼロを確認(下記コマンド)— docs/ + リポジトリ根README + ari-core ソース + HTML
[ ] 4. 受け入れゲート(§6)が削除「前」に exit 0
[ ] 5. 作業ブランチ上であり、直近コミットでリバート可能
[ ] 6. 削除はフェーズ末にまとめ、削除専用コミット(メッセージに対象と理由を明記)で行う
```

被参照ゼロの確認コマンド(対象パスを `OLD` に代入):

```bash
OLD='docs/guides/paperbench/paperbench_quickstart.md'        # 例: 削除/移動する旧パス
git grep -nF "$OLD"                               # 旧パスへの全参照(リポジトリ全体)
git grep -nF "$(basename "$OLD")"                 # ファイル名だけの参照も確認
# → ヒットが残る場合は追従漏れ。ゼロになるまで削除しない。
```

### 種別ごとの追加 double-check

**D-1(移動)**:
```bash
[ ] git mv を使用(cp+rm は禁止)
[ ] git log --follow -- <新パス> で履歴が連続(コピー扱いになっていない)
[ ] 移動先の内容が移動元と同一(git mv なら自明。手作業なら sha256 比較)
```

**D-2(空ディレクトリ)**:
```bash
[ ] find docs/howto -type f | wc -l  が 0
[ ] rmdir docs/howto   # 空でなければ失敗する=安全弁。-rf は使わない
```

**D-3(スタブ)**:
```bash
[ ] そもそもスタブを設置したか(未設置なら対象なし)
[ ] 旧パスへの被参照ゼロ(共通#3)
[ ] スタブ本文が「移動先ポインタ」のみで、固有コンテンツを含まない
```

**D-4(重複メディア)**:
```bash
[ ] sha256sum <旧> <新>  が一致
[ ] 旧パスへの参照(*.md / *.html / src=)がゼロ
[ ] 削除は1ファイルずつ。ディレクトリ一括 rm -rf は禁止
```

**D-5(計画書3点の削除)**:
```bash
[ ] 3点とも本エージェントが作成したもの(作成者確認)。第三者作成物が混ざっていない
[ ] 対応する scope が完了済(対応表・パリティ表・ゲートがすべて通過)
[ ] 他 doc / README / HTML から計画書3点への参照が残っていない(共通#3)
[ ] 削除は3点まとめて [plan-deletion] コミット1本(メッセージに復元手順を明記)
[ ] 復元手段の確認: git log --diff-filter=D --follow -- <path> で後から取り出せる
```

**D-6(stale 参照)**:
```bash
[ ] 当該パスが「削除された」か「移動/改名された」かを git log で判定
[ ] 移動/改名なら新パスへ付け替え(参照は残す)。本当に消滅した機能のみ参照削除
[ ] 削除後に該当 doc の主張が宙に浮かないか(根拠喪失)を確認
```

## 4. 受け入れゲート(統合)

各フェーズ末、かつ**あらゆる削除の前**に実行する。

1. **壊れたリンク=0**: 全 `.md` の相対リンク・`docs/`プレフィックスリンクが実在に解決。
2. **HTML href 解決=0破壊**: `index.html`/`docs.html` の `.md`・メディア href が実在。
3. **`~/.ari/` 残渣=0**(既存ゲート維持。`_archive/` 除外)。
4. **front-matter ソース実在=0欠落**: `python scripts/docs/check_doc_sources.py` が exit 0。
5. **パリティ表整合**: `docs/README.md` のパリティ表が実ファイルと一致(U4 後は全 ✓)。
6. **孤立ファイル=0**: `docs/README.md` 目次から全ライブ doc へ到達可能。
7. **移動履歴連続**: `git log --follow` でコピー&削除でなく移動であること。

`release_policy.md` の Release checklist §4 に追加する2項:
> - `python scripts/docs/check_doc_sources.py` が exit 0。
> - docs 内リンク検査スクリプトが exit 0(壊れたリンク=0)。

## 5. リスクとロールバック

| リスク | 緩和 |
|---|---|
| 公開HTMLサイトのナビ崩壊 | U3 で href 全件追従 + ゲート2。削除前 double-check #3 |
| 156リンク + 根README追従漏れ | `git grep` 一括検出 + ゲート1。削除は被参照ゼロ確認後 |
| 移動が cp+rm 扱いで履歴断絶 | `git mv` 限定 + ゲート7 |
| メディア誤削除(外部公開資産) | D-4 の sha256 一致 + 1ファイル単位削除。`rm -rf` 禁止 |
| stale 参照を「削除」して根拠喪失 | D-6: 移動/改名は付け替え、消滅のみ削除 |
| 計画書を早期削除し追跡不能 | D-5: scope 完了確認後にのみ削除。`[plan-deletion]` 専用コミット + `git log --diff-filter=D --follow` で復元可能 |

**ロールバック**: すべて `bfts` から切る作業ブランチ上で実施。フェーズ単位・削除専用の
コミット分割により、`git revert <コミット>` / `git restore` / ブランチ破棄で即時復旧。
削除専用コミットを分けることで「削除だけを戻す」が可能になる。
