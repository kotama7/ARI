# ari

Core engine package for ARI. Each sub-package carries its own `README.md`
(thin entry point) and an authoritative module docstring in `__init__.py`.

## Contents

- `README.md` тАФ this file.
- `__init__.py` тАФ package marker.
- `_deprecation.py` тАФ `DeprecationWarning` helpers for v0.5тЖТv1.0 legacy paths/aliases.
- `checkpoint.py` тАФ checkpoint JSON I/O (`tree.json` / `nodes_tree.json` / `results.json`).
- `cli_ear.py` тАФ `ari ear тАж` curation / publish / promote / status CLI surface.
- `container.py` тАФ unified container runtime abstraction (Docker / Singularity / bare subprocess).
- `core.py` тАФ generic ARI runtime builder (zero domain-specific code; `cli` calls into it).
- `cost_tracker.py` тАФ per-call logs + per-experiment cost summaries.
- `env_detect.py` тАФ detect schedulers, container runtimes, and HPC resources.
- `lineage.py` тАФ recursion lineage helpers; walk `parent_run_id` chains for ancestor artifacts.
- `memory_cli.py` тАФ `ari memory` subcommand (migrate / backup / тАж).
- `paths.py` тАФ centralised `PathManager` for directory layout/resolution.
- `pidfile.py` тАФ `.ari_pid` write/read/cleanup for run-liveness detection.
- `agent/` тАФ ReAct loop, environment capture, per-stage workflow guidance.
  - `README.md` тАФ agent index.
  - `__init__.py` тАФ package module-map docstring.

  - `metric_contract.py` тАФ producer/agent half of the metric-correctness contract (mirrors `pipeline.claim_gate`): domain-neutral obligation text (`build_contract_obligation`), run-level claim-coverage + lineage-chaining steering (`build_coverage_status`, `build_expand_coverage_hint`, `build_inherited_data_note`, `collect_node_measurement_names`), and the post-emit continuation nudge (`build_emission_nudge`).
  - `react_driver.py` тАФ generic ReAct driver for pipeline `react:` stages, with sandbox enforcement.
  - `run_env.py` тАФ capture/read helper for `_run_env.json`.
  - `tool_manager.py` тАФ OpenAI tool conversion, dispatch, phase-aware filtering.
  - `workflow.py` тАФ `WorkflowHints` dataclass injected into `AgentLoop`.
  - `shims/` тАФ executable `PATH` shims for the reproducibility sandbox.
    - `README.md` тАФ shims index.
    - `git.sh` тАФ intercepts only `git clone` of the paper's ref; other git passes through.

  - `README.md` тАФ evaluator index.

  - `deterministic_evaluator.py` тАФ `DeterministicEvaluator`: non-LLM judge owning the SpMM measurement; writes `metrics._scientific_score` to drive BFTS selection (handoff study B2). Selected via `ARI_EVALUATOR=deterministic`.
  - `dynamic_axes.py` тАФ venue/run-specific evaluation-axis derivation.
  - `handoff_stats.py` тАФ run-level analysis statistics for the handoff study (Stage 4 core): geomean, run-cluster bootstrap CI, TOST equivalence (RQ1 parity), Holm correction, per-arm summary. Pure; consumed by `scripts/analyze_handoff_ablation.py`.
  - `llm_evaluator.py` тАФ `LLMEvaluator`: extraction + multi-axis composite scoring.
  - `spmm_harness.py` тАФ SpMM measurement core (handoff study B2b): fp64 reference oracle, per-element correctness bound (eps model), seeded matrix families, geomean aggregation (`measure_node`). Pure parts login-tested; compile/run/timing runner is compute-node only.
  - `spmm_kernels/` тАФ C kernel fixtures for the handoff-study SpMM measurement (B2b). The frozen
    - `README.md` тАФ spmm_kernels index.
    - `baseline_spmm.c` тАФ FROZEN 1x reference CSR SpMM (naive, single-thread); the speedup denominator.
    - `candidate_spmm.c` тАФ agent-edited `spmm()` template (seeded identical to baseline); copied per node into the work_dir.
    - `experiment.md` тАФ the SpMM optimization task handed to the agent (goal + `spmm()` contract + the deterministic-evaluator judging rules); the experiment file for the pilot/MVP runs.
    - `Makefile` тАФ manual build (the Python runner compiles directly with identical flags for baseline/candidate).
    - `Plan.md` тАФ B2b compile/run/timing runner plan (deps + deletion requirement).
    - `spmm_kernel.h` тАФ the `spmm()` contract the candidate must keep.
    - `spmm_main.c` тАФ FROZEN timing + binary-I/O harness (warmup/reps/median); agent must not edit.
